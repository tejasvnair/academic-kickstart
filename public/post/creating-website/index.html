This month, I finally managed to convince myself to create a website. Some tools made this really easy and that's what this post is all about.

Choosing a site generator

Options like Wordpress, Medium make writing/publishing really easy for non-programmers who just want to publish their content and don't care about retaining more control over their content.

Being a programmer, I preferred to create a website from scratch and after exploring some options, narrowed it down to static site generators (SSGs). This article goes into detail about SSGs.

Among the SSGs, Jekyll was my initial choice but Ruby dependencies made it very limiting.

Hugo, on the contrary, is based on a single binary and builds sites in milliseconds. I could preview the content updates in (almost) real-time!

Template for website

I chose the Academic theme for Hugo which made it really easy to create a minimal website. It's a great framework for writing code, math, and other content.

For version control of the content, GitHub was an easy choice.

Hosting

I chose Netlify instead of GitHub Pages for hosting the website since it supports continuous deployment with Hugo. This post makes the case for choosing Netlify over GitHub Pages.

Domain registration

There are a variety of domain registrars to choose from. Here's an article comparing the top registrars. I chose NameCheap to register a personal domain and Netlify made it easy to setup a custom DNS. This article details the steps for setting up a custom DNS with illustrations.

Summary

In brief, this is what went into creating this website:

  • Hugo + Academic theme for content creation and building site
  • Github for version control of website content
  • Netlify for hosting the website
  • NameCheap for domain registration
